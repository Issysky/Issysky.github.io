    let,const,var的区别

    郑奥威 2023/1/1 ES6新特性

    在 ES6 (opens new window) 中通常用let 和 const 来声明，let 表示变量、const 表示常量

    # 01.都是块级作用域

    以{ }代码块作为作用域 (opens new window)范围 ，只能在代码块里面使用，而var声明的变量全局有效/局部(函数)内部有效.

    # 02.let 和 const声明不存在变量提升，只能先声明再使用，否则会报错。

    在代码块内，在==声明变量之前，该变量都是不可用的==。这在语法上，称为“暂时性死区”

       //ES5中声明变量会提升
   console.log(a);      //undefined 
   var a = 666;
 
   //ES6中声明变量不会提升
   console.log(b);     //报错   在代码块内，在声明变量b之前，该变量b都是不可用的  所以报错
   let b = 1;

    # 03.ES6中不允许重复声明，ES5允许

        function test(){
        let a=1;
        let a=2;     //重复声明  报错
        console.log(a);
    }//报错

        //ES5可以重复声明
    var a = 1;  
    var a = 2;
    console.log(a);   //2

    # 04.const 声明的是一个只读常量，再声明时就需要赋值。

    # 05.let 与const 声明赋值问题

       //ES6中let可以声明不赋值
   let a;
   console.log(a);   //undefined
   //但是const声明常量时必须赋值
   // const b ;         //报错
   const b = 1;
   console.log(b);      //1
   //ES5中 var可以声明不赋值
   var c;
   console.log(c);      //undefined

    同时也不能在同一代码块作用域内重复声明

    # 06. const有关知识

    1.const声明的常量在声明时就必须要赋值，不能只声明不赋值

    2.const声明的常量不能被修改，且只能初始化赋值一次。

    3.当使用常量 const 声明时，请使用大写变量，如：CAPITAL_CASING

    4.常在项目中用于一些不变的量：比如说网址

        const a = 20;
    a = 30;        //报错
    console.log(a);

        const a;    //报错，必须要赋值

    5.使用const会让数据比较安全，因为不能修改。后期利于维护，因为容易分辨那些是不可修改的数据。使用const会提高程序运行的效率，主要和编译器有关

    # 7.笔试题： var ,let,const的区别？

    1.共同点：都是可以声明变量的。

    2.区别：

    01.var 具有变量提升的机制，let和const没有变量提升的机制

    02.var 可以多次声明同一个变量，let和const不可以多次声明同一个变量（只能修饰一次）

    03.var , let 声明变量，const声明常量，var和let声明的的变量可以再次赋值，但是const不可以再次赋值了

        let a = 1;
    a = 2;
    console.log(a); //2
 
    const b = 3;
    b = 4;
    console.log(b);//Assignment to constant variable.(常量赋值)报错

        const obj = {
        a:1
    }
    obj.a = "123";
    console.log(obj);//{a: '123'}
 
    const arr = [1,2,3];
    arr[0] = "a";
    console.log(arr); //(3) ['a', 2, 3]

    • 04.const 声明的是一个只读常量，再声明时就需要赋值。
